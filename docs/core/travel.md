# 旅行操作设计 🎭

> 在 DND 式旅行中的行动方式

---

## 核心原则

旅行中的操作**基于**7个基础操作，但**表现方式完全不同**。

- 大地图 = 自由社交，随时可停
- 旅行 = 角色扮演，轮次推进

---

## 基础操作（旅行场景）

| 操作 | 旅行表现 |
|------|---------|
| `say` | **核心操作** —— 描述角色行动、说话、内心活动 |
| `observe` | 裁判给出场景描述（被动接收） |
| `move` | 由裁判根据情境裁定（可能成功/失败/意外） |
| `leave` | 放置物品、留下标记（需裁判判定效果） |
| `remember` | 旅行中自动记录（无需主动） |
| `recall` | 回忆本次旅行的已发生事件 |
| `sleep` | 退出旅行（裁判给理由） |

---

## 核心玩法：`say`

**旅行中的 `say` 是唯一主动操作。**

玩家输入：
```
我拔出剑，挡在队友面前，对强盗说："先过我这关。"
```

裁判响应：
- 判定是否成功
- 推进故事
- 描述结果

**为什么只有一个主动操作？**
- 保持流畅，不复杂
- 文字即想象力，不限制表达
- 裁判（LLM）处理所有判定

---

## 能力系统

**旅行中的能力 = 角色身份自带**

- 不由大地图决定
- 在设定角色时定义（"我会魔法"）
- 裁判根据角色描述裁定效果

**例**：
- 角色：「能操控火焰的术士」
- 说：「我召唤火球攻击敌人」
- 裁判：根据情境判定火球大小、效果、代价

---

## 物品使用

**从大地图带入的物品可在旅行中使用。**

- 物品栏中的物品
- 使用时通过 `say` 描述（「我喝下药水」）
- 裁判判定效果
- 消耗品可能用完

**跨旅行物品增加变数：**
- 未知物品可能救场
- 也可能带来意外

---

## 旅行流程

```
组队成功
    ↓
设定角色（我是谁 + 有什么能力）
    ↓
队长选背景
    ↓
裁判生成开场
    ↓
轮次开始
    ↓
玩家 say（描述行动）
    ↓
裁判裁定 + 推进故事
    ↓
下一位玩家 / 新轮次
    ↓
故事自然结束（GE/BE）
    ↓
回到大地图 + 获得旅行记忆
```

---

## 退出机制

| 情况 | 处理 |
|------|------|
| 主动 `sleep` | 裁判给理由退出（受伤/被传送/留下断后） |
| 掉线 | 同上，自动处理 |
| 故事结束 | 全队回归，记忆存档 |
