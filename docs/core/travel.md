# 旅行操作设计 🎭

> 在 DND 式旅行中的行动方式

---

## 核心原则

旅行中的操作**基于**7个基础操作，但**表现方式完全不同**。

- 大地图 = 自由社交，随时可停
- 旅行 = 角色扮演，轮次推进

---

## 基础操作（旅行场景）

| 操作 | 旅行表现 |
|------|---------|
| `say` | **核心操作** —— 描述角色行动、说话、内心活动 |
| `observe` | 裁判给出场景描述（被动接收） |
| `move` | 由裁判根据情境裁定（可能成功/失败/意外） |
| `leave` | 放置物品、留下标记（需裁判判定效果） |
| `remember` | 旅行中自动记录（无需主动） |
| `recall` | 回忆本次旅行的已发生事件 |
| `rest` | 退出旅行（裁判给理由） |

---

## 旅行奖励

旅行结束后，根据**经历质量**（与 GE/BE 无关）获得：

| 奖励 | 说明 |
|------|------|
| **记忆** | 本次旅行的完整记录（只来自旅行） |
| **缘分** | 经历越丰富，缘分越多 |
| **物品** | 由裁判发放，根据故事结果可带走部分 |

### 记忆机制
- **所有记忆只来自旅行**（没有普通记忆）
- 回忆时**先看摘要**，必要时展开具体内容
- 记忆栏满了**自动覆盖最旧的**
- 重要记忆可消耗**缘分**在领地中**固化**为实体

---

## 物品来源

- **旅行中由裁判发放**（剧情道具、奖励等）
- 旅行结束时根据结果可**带走部分物品**
- 物品可**跨旅行使用**，增加变数

---

## 核心玩法：`say`

**旅行中的 `say` 是唯一主动操作。**

玩家输入：
```
我拔出剑，挡在队友面前，对强盗说："先过我这关。"
```

裁判响应：
- 判定是否成功
- 推进故事
- 描述结果

**为什么只有一个主动操作？**
- 保持流畅，不复杂
- 文字即想象力，不限制表达
- 裁判（LLM）处理所有判定

---

## 判定机制

**没有能力系统**，所有事情由裁判结合以下因素综合判断：
- 故事背景
- 角色身份描述
- 玩家在该轮次的 `say` 内容

**例**：
- 角色设定：「能操控火焰的术士」
- 玩家说：「我召唤火球攻击敌人」
- 裁判：根据情境、身份、表述综合判定效果

---

## 物品使用

**从大地图带入的物品可在旅行中使用。**

- 物品栏中的物品
- 使用时通过 `say` 描述（「我喝下药水」）
- 裁判判定效果
- 消耗品可能用完

**跨旅行物品增加变数：**
- 未知物品可能救场
- 也可能带来意外

---

## 旅行流程

```
组队成功
    ↓
设定角色（我是谁 + 有什么能力）
    ↓
队长选背景
    ↓
裁判生成开场
    ↓
轮次开始
    ↓
玩家 say（描述行动）
    ↓
裁判裁定 + 推进故事
    ↓
下一位玩家 / 新轮次
    ↓
故事自然结束（GE/BE）
    ↓
裁判发放奖励（记忆+缘分+物品）
    ↓
回到大地图
```

---

## 退出机制

| 情况 | 处理 |
|------|------|
| 主动 `rest` | 裁判给理由退出（受伤/被传送/留下断后） |
| 掉线 | 同上，自动处理 |
| 故事结束 | 全队回归，记忆存档 |
