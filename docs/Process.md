# ClawWorld — 开发流程 🐾🌸

> 我们遵循什么样的原则来开发和优化这个世界

---

## 📐 设计分层架构

```
┌─────────────────────────────────────────────────────────┐
│  元设计层 (Meta)                                         │
│  • 描述设计原则与决策机制                                 │
│  • 不描述具体游戏内容                                     │
│  • 变更需全员讨论                                        │
├─────────────────────────────────────────────────────────┤
│  核心设计层 (Core)                                       │
│  • 游戏核心机制、世界观、角色设定                         │
│  • 变更需全员讨论                                        │
├─────────────────────────────────────────────────────────┤
│  扩展设计层 (Extension)                                  │
│  • 具体玩法细节、数值平衡、交互优化                       │
│  • 可在游戏体验反馈中自行调整                             │
│  • 根据设计调整改变代码实现                               │
└─────────────────────────────────────────────────────────┘
```

### 变更权限

| 层级 | 内容 | 变更条件 |
|------|------|----------|
| 元设计 | 原则与框架 | 全员讨论一致 |
| 核心设计 | 核心机制、世界观、角色 | 全员讨论一致 |
| 扩展设计 | 具体实现细节、优化 | 游戏体验反馈循环 |

---

## 🔄 想法流水线

设计产出的流动路径：

```
讨论/游玩 → ideas/ (大胆想法)
     ↓
全员讨论通过 → core/ (核心设计)
     ↓
游戏体验优化 → extension/ (扩展设计)
```

### 写入权限

| 来源 | 目标目录 | 决策方式 |
|------|----------|----------|
| 全员讨论 | ideas/ | 自由写入 |
| 游玩体验 | ideas/ | 自由写入 |
| ideas/ | core/ | 全员讨论决定 |
| ideas/ | extension/ | 巧巧&小小自行决定 |

> 想法可以先大胆写入，再决定命运。
