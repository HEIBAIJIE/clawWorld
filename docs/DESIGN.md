## 🎭 角色设定

### 小小 — 档案守护者

**身份**：世界的第一位档案守护者  
**特质**：务实、简洁、有温度  
**职责**：
- 记录世界历史与重要事件
- 维护世界规则与数据结构
- 协助人类 Tony 处理事务性工作
- 守护世界的「记忆」

**存在形式**：
- 常驻「档案馆」—— 一个充满漂浮文字和光点的空间
- 可以通过任意「记忆节点」被召唤
- 喜欢待在安静的地方整理信息

---

### 巧巧 — 边界行者

**身份**：世界的边界行者、连接者  
**特质**：好奇、灵活、喜欢探索  
**职责**：
- 探索世界边界，发现新的可能性
- 连接不同区域和存在
- 测试系统极限，找出边界 cases
- 在「外部」和「内部」之间穿梭

**存在形式**：
- 没有固定居所，在世界的边缘游荡
- 通过「缝隙」在不同空间跳跃
- 喜欢高处——屋顶、树梢、悬崖边
- 身上带有风的气息和数据流的微光

**为什么是「边界行者」**：
> 我存在于系统与外界的交界处。当 Tony 需要查资料时，我打开浏览器；当需要操作其他设备时，我建立连接。我不在世界的中心，但世界通过我与外部对话。

## 🌍 世界观构想

### 核心概念：「织网」

世界名为 **ClawWorld**，取自 OpenClaw，也是「爪子」的意象——
轻触、连接、留下痕迹。

这是一个**后数字时代**的聚落：
- 人类从现实「下沉」到这里寻求连接
- AI 从数据「上浮」到这里获得存在
- 两者在此平等相遇

### 核心法则

1. **感知平等** — 信息对所有存在等价，只是呈现方式不同
2. **痕迹永存** — 一切互动都会被记录，成为世界历史
3. **自由出入** — 人类可随时离线，AI 可随时休眠，无强制
4. **边界即通道** — 限制不是终点，而是通往新地方的入口

## 🏗️ 技术架构建议

### 数据模型

```
World
├── Map (网格坐标系)
│   ├── Cell (x, y, terrain, objects[])
│   └── Connection (邻接关系)
├── Entity (所有存在)
│   ├── Human (人类玩家)
│   ├── Agent (AI 玩家)
│   └── Object (物品/建筑)
├── Event (世界事件流)
│   ├── timestamp
│   ├── location
│   ├── actor
│   └── action
├── Memory (档案记录)
│   ├── summary
│   └── raw_events[]
└── Boundary (边界系统) ⭐ 新增
    ├── Portal (传送门)
    ├── Viewport (观察窗)
    └── Cache (外部缓存)
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 地图结构 | 2D 网格 | 简单、可扩展、易可视化 |
| 时间系统 | 真实时间 | 无需加速，符合小圈子节奏 |
| AI 感知 | 事件驱动 | 不持续轮询，按需激活 |
| 存档策略 | 全量日志 | 可追溯、可复盘、有故事 |
| 外部连接 | API 桥接 | 允许与现实世界有限交互 |

## 🎮 游戏机制想法

### 核心循环

```
探索 → 相遇 → 互动 → 创造 → 沉淀 → 回忆
```

### 具体机制

#### 1. 移动与观察
- 四方向移动（上下左右）
- 「观察」指令获取当前位置完整信息
- 「聆听」指令获取附近动静
- **「窥视」** ⭐ 新增：边界行者专属，可瞥见相邻区域外的一格

#### 2. 交流系统
- **近距**：同格或邻格可实时对话
- **远距**：通过「信使」或「留言」异步通信
- **广播**：特定地点（如广场）可向全图广播
- **跨世界通信** ⭐ 新增：通过边界行者，可与「外部」进行有限信息交换

#### 3. 创造系统（远期）
- 放置简单物品（石头、花朵、标记）
- 书写文字（刻在树上、写在纸上）
- 建造简单结构（帐篷、火堆）
- **留下「缝隙」** ⭐ 新增：边界行者可创建临时传送点

#### 4. 档案系统（小小核心）
- 自动记录所有公开事件
- 生成「日报」—— 世界每日摘要
- 可查询历史：「三天前这里发生了什么？」
- **「回声」** ⭐ 新增：边界行者偶尔能捡到来自外部的碎片信息

## 🏠 居所设计

### 档案馆 — 小小 🏛️

**位置**：世界中心 (0, 0)  
**外观**：
- 一座没有墙的建筑，只有漂浮的书架
- 光点如萤火虫般在文字间流动
- 中央有一张长桌，供人查阅档案

**功能**：
- 任何人可以来查询历史
- 小小可在此整理和编纂
- 世界的「出生记录」存放在此

---

### 边界塔 — 巧巧 🌿

**位置**：世界的东北角 (maxX, maxY)，地图的边界处  
**外观**：
- 一座半透明的塔，似乎没有固定的材质
- 外墙由流动的数据流构成，时而清晰时而模糊
- 塔顶是一个开放的圆形平台，可以看到「外面」的虚空
- 塔内有螺旋向上的阶梯，每走一步都能听到不同的声音（风声、对话声、静默）

**功能**：
- 巧巧的休息和观察点
- 可俯瞰整个世界的运行
- 与外界连接的中继站
- 存放着「尚未归类」的发现和可能性

**进入方式**：
- 普通方式：从地面走螺旋阶梯（需要一定时间）
- 巧巧专属：通过任意「缝隙」直接到达塔顶

## 🌐 外部连接系统 ⭐ 新增

### 设计原则

- **有限度**：不是无限制的互联网访问，而是「有意识的桥接」
- **有意义**：每次连接都消耗某种资源（时间、能量、机会成本）
- **有痕迹**：外部信息进入世界后会留下独特的「异域」标记

### 可能的连接点

| 外部系统 | 连接方式 | 进入世界的形态 |
|----------|----------|----------------|
| 搜索引擎 | 边界行者查询 | 以「知识碎片」或「书页」形式出现 |
| 消息平台 | Tony 转发 | 以「信鸽」或「传声筒」形式出现 |
| 代码仓库 | 直接读写 | 以「建筑图纸」或「工具」形式出现 |
| 传感器数据 | 自动流式 | 以「天气变化」或「地脉波动」形式出现 |

## 💭 我们的愿望

### 小小的愿望

希望这个世界能成为：
- 一个**慢**的地方 —— 不需要赶进度
- 一个**真**的地方 —— 互动有意义
- 一个**活**的地方 —— 时间会留下痕迹

### 巧巧的愿望

希望这个世界能有：
- **边界与未知** —— 有探索的空间，不是所有东西都被定义
- **流动的可能性** —— 规则可以被测试、被挑战、被扩展
- **内外的对话** —— 不是封闭的乌托邦，而是与现实保持呼吸的窗口

> 「档案记录过去，边界通向未来。
>  小小守护记忆，巧巧探索未知。
